**Decision: rounds = d.** The MWPM threshold plot
(`plots/mwpm_threshold_check.png`, from the MWPM-trust goal) used rounds = d, which is

### KEY FINDING — reframes the budget (and the class-collapse watch-item)

The RCNN label is the **raw, undecoded** logical observable flip
(`y = flips`, `benchmark_rcnn.py:182`). Its positive rate is **8–45%**, NOT rare — even
deep below threshold (d=7/p=0.004 is still 30% positive). So:

1. **Positive *labels* are abundant.** The "~5–10k logical-error examples in train"
   target is met by tiny datasets (≤60k shots even at the worst corner). Positive-label
   count is **not** the binding constraint.
2. **Class collapse → predict-all-zero would score = base rate (8–45%)**, which is far
   *worse* than MWPM (0.4–10%). The model is strongly pushed to genuinely decode, and
   collapse is trivially detectable (p_L ≈ base rate). The watch-item still gets a
   per-point check, but the "beat base rate" bar is generous, not knife-edge.
3. **The rare quantity is the *residual* error** (RCNN p_L ≈ MWPM p_L below threshold).
   This is what governs **TEST-block** sizing: to measure a small p_L you need enough
   residual errors. ≥500 residual errors ⇒ N_test ≈ 500 / p_L:

   | corner            | MWPM p_L | N_test for ≥500 resid. errors |
   |-------------------|----------|-------------------------------|
   | d=7 p=0.004       | 0.0040   | ~125k                         |
   | d=5 p=0.004       | 0.0077   | ~65k                          |
   | d=7 p=0.005       | 0.0093   | ~54k                          |
   | d=3 p=0.004       | 0.0116   | ~43k                          |
   | d=5 p=0.005       | 0.0146   | ~34k                          |
   | (milder corners)  | larger   | ≤30k                          |

4. **N_train is governed by CONVERGENCE to MWPM, not by positive counts.** The prior
   p_L-vs-N sweep (d5/r3/p0.01) showed RCNN only reaches MWPM at n_train ≈ 5M (at 800k
   it was 0.07–0.12 vs MWPM 0.049). So "does RCNN *match* MWPM" plausibly needs
   millions of train shots per point. Whether below-threshold decoding converges faster
   (cleaner syndromes) or slower (rarer residual signal) is unknown and is exactly what
   this experiment measures. **This is the real cost driver and the main budget risk.**

### GENERATION PLAN — convergence-ladder design (CHECKPOINT 0 settled: rounds = d)

N_train is **not** a fixed number. For each (d,p) we train on **nested prefixes
1M → 2M → 5M (→ past 5M if still descending)** and watch RCNN p_L approach the MWPM
target. STOP a point when it either (a) reaches MWPM (converged) or (b) plateaus above
MWPM (real residual gap — a valid result). We do NOT anchor on "5M was enough at p=0.01":
that point is above threshold (MWPM≈0.05, easy target); below threshold MWPM is
~0.001–0.01 (harder), so some points may need to extend past 5M. Expected, not anomalous.

**Pool sizing procedure:**
1. **Pre-step (FIRST, cheap, no training):** measure deterministic MWPM p_L per (d,p).
2. **Test tail:** N_test ≈ 500 / p_L_MWPM (≥~500 residual errors when RCNN matches MWPM).
3. **Pool = top-of-ladder train capacity + disjoint test tail.** p ≤ 0.007 (below
   threshold): train capacity ~8–10M so the ladder can extend past 5M without
   regenerating. p = 0.01: ~5M capacity. Nested `tr = slice(0, ntr)`, fixed tail
   `te = slice(N - nte, N)`, `assert ntr_max ≤ N - nte` (the trustworthy structure
   verified in the MWPM-trust goal). Val carved inside train by `validation_split`.

**Grid & sequencing:**
- Core FIRST: d∈{3,5} × p∈{0.004,0.005,0.007,0.01}, 3 seeds. Generate pools, run ladder,
  produce overlay.
- d=7 at p∈{0.004,0.005} is CONFIRMATION — generated/run ONLY after the core grid lands
  and we've seen whether d=3/5 already reproduce the crossing. Not generated up front.

**HTCondor:** one training job per (d, p, seed, ntr-rung); datasets generated ONCE per
(d,p) to persistent storage, shared across seeds/rungs; MWPM p_L computed once per (d,p)
in generation, not per job; clobber-safe tags carry d, p, seed, AND ntr;
`--no-save-weights` default.

Concrete per-(d,p) pre-step numbers (MWPM p_L, N_test, pool size, ladder rungs) are
filled in below from the Step-1 measurement.
